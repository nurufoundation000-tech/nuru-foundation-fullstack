1. What is an Exploit?

🤔 WHAT is an Exploit?

An exploit is a technique, script, or piece of code used to take advantage of a vulnerability in a system. Think of it as the "key" that fits a specific "lock" (the vulnerability). Without the right key, you can't open the door.

🔓 Vulnerability

A weakness in a system. Could be a bug in software, a misconfiguration, or a design flaw.

Example: An outdated FTP server with a known buffer overflow.

⚡ Exploit

The method used to take advantage of the weakness. This is the actual attack code or technique.

Example: A Python script that sends specially crafted packets to trigger the buffer overflow.

📦 Payload

The code that runs after successful exploitation. This is what you actually want to happen.

Example: A reverse shell that gives you remote access to the target.

📌 Real-World Example:

A server is running vsftpd 2.3.4 (an outdated FTP service).

  • The outdated version contains a known backdoor vulnerability (CVE-2011-2523)
  • An attacker uses a public exploit targeting that specific version
  • The exploit triggers the backdoor and opens a reverse shell
  • The attacker now has remote access to the server
📌 Types of Exploits
  • Remote Exploits: Attacker does not need prior access. Can exploit over the network.
  • Local Exploits: Attacker already has limited access and wants to escalate privileges.
  • Client-side Exploits: Target user applications (browser, PDF reader, Office). Requires user interaction.
📌 Important Concept

Exploitation is only possible when ALL of these conditions are met:

  • A vulnerability exists in the target
  • The exploit matches that specific vulnerability
  • The target system is reachable (network connectivity)
  • The exploit doesn't crash the target before succeeding

2. Metasploit Framework (Detailed Introduction)

🤔 WHAT is Metasploit?

Metasploit is a professional exploitation framework used by penetration testers and security researchers. It simplifies the exploitation process by providing ready-to-use modules for thousands of vulnerabilities.

Think of it as a "weapon store" for security testing — but only use it in your own lab!

📦 Components of Metasploit
  • Exploit: The code that targets a specific vulnerability
  • Payload: Code that executes after exploitation (reverse shell, bind shell, etc.)
  • Auxiliary Modules: Scanning, enumeration, brute force (no payload)
  • Encoders: Modify payloads to avoid antivirus detection
  • Post Modules: Actions after exploitation (privilege escalation, persistence)
🎯 Types of Payloads
  • Reverse Shell: Target connects BACK to attacker. Most common. Bypasses firewalls.
  • Bind Shell: Target opens a port. Attacker connects to it. Often blocked by firewalls.
  • Meterpreter: Advanced Metasploit payload with many features (file upload, keylogging, etc.)
# Starting Metasploit
msfconsole
# Core Workflow (6 steps)
1. search <service> # Find exploit for your target
2. use <module> # Select the exploit
3. show options # See what needs to be configured
4. set RHOST <target-ip> # Set target IP
5. set LHOST <your-ip> # Set your IP for reverse shell
6. set PAYLOAD <payload> # Choose what happens after exploit
7. run or exploit # Execute!
💡 Pro Tip: Always run show options before exploiting. Many beginners forget to set required variables (RHOST, LHOST, RPORT).

3. Exploiting Simple Vulnerabilities (Step-by-Step Logic)

🤔 HOW does exploitation work in practice?

Exploitation is a structured, repeatable process — not guessing. You follow a logical sequence from reconnaissance to access.

📋 The 6-Step Exploitation Process
  1. Reconnaissance: Identify open ports, services, and versions using Nmap: nmap -sV <target-ip>
  2. Identify Vulnerability: Match service version with known vulnerabilities (searchsploit, CVE databases)
  3. Select Exploit: Find matching exploit in Metasploit or exploit-db.com
  4. Configure Exploit: Set target IP, payload, local IP for reverse connection
  5. Execute: Run the exploit and observe the response
  6. Access: If successful, you get a shell or Meterpreter session
🎯 Example: vsftpd 2.3.4 Backdoor
# Step 1: Reconnaissance
nmap -sV 192.168.56.102
21/tcp open ftp vsftpd 2.3.4
# Step 2-3: In Metasploit
search vsftpd
use exploit/unix/ftp/vsftpd_234_backdoor
set RHOST 192.168.56.102
set PAYLOAD cmd/unix/interact
run

Key Understanding: Exploitation is not guessing. It's based on accurate information and correct matching of vulnerabilities to exploits.

✅ What happens when an exploit succeeds?

The payload executes. For a reverse shell, the target connects back to your machine, and you get command-line access. You can then run commands like whoami, ls, id to confirm access.

4. Privilege Escalation

🤔 WHAT is Privilege Escalation?

Privilege escalation is the process of increasing access rights after initial compromise. You start with limited access (e.g., a low-privilege user) and move to higher privileges (administrator/root).

Why it's needed: Initial access is often limited. Full control of a system requires admin/root access.

📌 Types of Privilege Escalation
  • Vertical Escalation: Normal user → Administrator/Root (most common goal)
  • Horizontal Escalation: User A → User B (accessing another user's data)
🔧 Common Escalation Techniques
  • Misconfigured sudo permissions
  • SUID binaries (programs that run as root)
  • Weak file permissions (writable config files)
  • Outdated kernel vulnerabilities
  • Writable system files (/etc/passwd, /etc/sudoers)
  • Cron jobs running as root with writable scripts
# Step 1: Enumeration (CRITICAL before escalation)
whoami # What user am I?
id # What groups am I in?
uname -a # What kernel version? (known exploits?)
sudo -l # What sudo commands can I run?
find / -perm -4000 2>/dev/null # Find SUID binaries
🎯 SUID Exploitation Example:

SUID (Set User ID) binaries run with the permissions of the file OWNER, not the user executing them. If a binary owned by root has SUID set, anyone running it gets root privileges temporarily.

# Find SUID binaries
find / -perm -4000 -type f 2>/dev/null
# Example dangerous SUID binary: /usr/bin/pkexec
# If vulnerable version exists (CVE-2021-4034), can escalate to root

Key Concept: Privilege escalation depends on finding weaknesses in system configuration — not just software vulnerabilities.

🔍 Privilege Escalation: Essential Enumeration Commands

Before attempting any escalation, you MUST gather detailed information about the target system.

🖥️ System Information
uname -a # Kernel version (check for exploits)
cat /etc/os-release # Distribution version
hostname # Machine name
cat /etc/issue # OS info
👤 User & Group Info
whoami # Current user
id # User ID, group IDs
cat /etc/passwd # All users on system
sudo -l # What sudo commands can I run?
📂 File System Enumeration
find / -perm -4000 2>/dev/null # SUID binaries
find / -perm -2000 2>/dev/null # SGID binaries
find / -writable -type f 2>/dev/null # Writable files
ls -la ~/.ssh # Check for SSH keys
🔌 Network & Services
netstat -tulpn # Listening ports
ps aux # Running processes
crontab -l # User's cron jobs
cat /etc/crontab # System cron jobs
💡 Automated Enumeration Tools: For faster enumeration, use tools like linpeas.sh, linux-exploit-suggester, or enum4linux. But first learn to do it manually!
