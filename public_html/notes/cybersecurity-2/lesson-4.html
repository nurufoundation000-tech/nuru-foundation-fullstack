1. Shell Mastery (Bash & Zsh)

A shell is a program that allows a user to interact with the operating system using commands. In cybersecurity, the shell is one of the most powerful tools because most tasks are performed through command-line interfaces.

🐚 Common shells
  • Bash (Bourne Again Shell) – default on most Linux distros
  • Zsh (Z Shell) – extended with plugins, used by pros
📌 Basic Command Structure

command [options] [arguments]

Example: ls -l /home

pwd → Shows current directory
ls → Lists files and directories
cd → Changes directory
mkdir → Creates a directory
rm → Deletes files
cp → Copies files
mv → Moves or renames files
cat → Displays file content
nano → Opens file editor
🔗 Command chaining: && (runs next if previous succeeds) · ; (runs sequentially).
Example: mkdir test && cd test

Importance in Cybersecurity: automation, penetration testing, system control and analysis.

2. Bash Scripting Basics

A Bash script is a file containing a sequence of commands that are executed automatically — force multiplier for any pentester.

📜 Basic Script Structure
#!/bin/bash
echo "Hello, World"
📦 Variables
name="Daniel"
echo $name
⚖️ Simple Conditions
if [ $name = "Daniel" ]; then
echo "Access granted"
fi
🔄 Loops
for i in {1..5}
do
echo $i
done
🎯 Importance: Automates repetitive tasks, reconnaissance, scanning — saves hours during penetration testing.

3. Permissions, Sudo, and SSH

🔐 File Permissions

Each file has three types of permissions: Read (r), Write (w), Execute (x).

ls -l → view permissions
chmod +x script.sh → make executable
👑 Sudo – Superuser Do

sudo allows a user to execute commands with administrative privileges.

sudo apt update
🔌 SSH (Secure Shell)

SSH is used to connect to remote systems securely. Essential for remote access and pivoting.

ssh user@192.168.1.10

Importance: remote system access, secure communication, heavily used in real-world environments and red teaming.

4. Log Files & Log Analysis

Log files record system activities. They are crucial for detecting attacks, monitoring systems, and investigating incidents.

📂 Common Log Location
/var/log/
  • auth.log → Login attempts
  • syslog → System events
  • kern.log → Kernel activities
🔍 Viewing & Searching Logs
cat /var/log/auth.log
grep "failed" /var/log/auth.log

Use tail -f for real-time monitoring.

🚨 Importance for Blue & Purple Teams:
• Detect unauthorized access
• Identify suspicious activity (brute force, privilege escalation)
• Support forensic investigations
