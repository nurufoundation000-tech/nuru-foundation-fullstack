Network Configuration & Snapshots

🎯 Objective: Connect your lab machines safely and protect your work

Two critical skills: making your VMs talk to each other (safely) + saving your progress so you never lose work.

📡 Understanding Host-Only Networking

🤔 WHAT is a Host-Only Network?

A Host-Only network is a private, isolated network that exists ONLY between your host computer and the virtual machines you create. Traffic on this network cannot reach the internet, and the internet cannot reach it.

❓ WHY do we need this?
  • Safety: Metasploitable is intentionally vulnerable. If connected to the internet, hackers could compromise YOUR computer or attack others.
  • Isolation: Your lab attacks stay contained — they won't accidentally hit real websites or devices on your home network.
  • Communication: Kali and Metasploitable need to "see" each other to simulate attacks. Host-Only creates a private playground.
  • Legal protection: You never accidentally scan or attack systems you don't own.
🔧 HOW does it work?
  • VirtualBox creates a virtual "network cable" connecting your VMs
  • IP addresses are assigned automatically (usually 192.168.56.x)
  • Your host computer gets 192.168.56.1 (the "gateway")
  • Kali might get 192.168.56.101, Metasploitable 192.168.56.102
  • They can ping each other, but NOT access Google.com or your home WiFi devices
⚠️ Important Warning: Never set Metasploitable to "Bridged" or "NAT" mode. That would give it internet access — extremely dangerous! Always use "Host-Only" for vulnerable machines.

🔧 Lab 1: Configure Host-Only Network

What we're doing: Creating a private, isolated network between your Kali and Metasploitable VMs so they can communicate safely.

📋 Step-by-Step Instructions
  1. Open VirtualBox — the main window showing your VMs
  2. Create Host-Only Network: File → Preferences → Network → Host-Only Networks → Click the "+" icon (green plus). This creates vboxnet0.
  3. Configure Kali VM: Right-click Kali VM → Settings → Network → Adapter 1 → Enable Network Adapter → Attached to: "Host-Only Adapter" → Name: "vboxnet0" → OK
  4. Configure Metasploitable VM: Repeat step 3 for Metasploitable VM
  5. Start both VMs — boot them up normally
  6. Find IP addresses: In each VM, open terminal and type ip a (or ifconfig). Look for "eth0" or "enp0s3" — the IP should be 192.168.56.x

📋 Step-by-Step (macOS)
  1. VirtualBox (Intel Macs): Same as Windows steps above
  2. UTM (Apple Silicon M1/M2/M3): Open UTM → Select VM → Edit → Network
  3. Change Network Mode to "Shared Network" — this creates an isolated network between UTM VMs
  4. For multiple VMs: UTM Preferences → Network → Create "Shared Network" → Assign both VMs to it
  5. Start both VMs → Run ip a to see their IP addresses

📋 Step-by-Step (Linux)
  1. VirtualBox: Same Windows steps apply
  2. KVM/QEMU: Open virt-manager → Select VM → Add Hardware → Network
  3. Create isolated network: virsh net-define and virsh net-start isolated
  4. Assign both VMs to the same isolated network
  5. Start VMs → ip a to verify IPs (usually 192.168.122.x)

📡 Testing Your Network Connection

What we're doing: Verifying that Kali and Metasploitable can "see" each other by sending test packets (pings).

✅ From Kali Linux (Attacker Machine)
ping 192.168.56.102 # Replace with Metasploitable's actual IP

What you should see:

64 bytes from 192.168.56.102: icmp_seq=1 ttl=64 time=0.5 ms
64 bytes from 192.168.56.102: icmp_seq=2 ttl=64 time=0.4 ms

This means: Your VMs are connected! Press Ctrl+C to stop pinging.

🔧 If Ping Fails (Troubleshooting)
  • No IP address? On Metasploitable, run: sudo dhclient eth0 to request an IP
  • Firewall blocking? On Metasploitable: sudo ufw disable
  • Wrong network? Check both VMs are on Host-Only, not NAT or Bridged
  • Network restart: sudo systemctl restart networking
  • Discover IPs: nmap -sn 192.168.56.0/24 from Kali
💡 Understanding the Ping Command:

Ping sends ICMP Echo Request packets. If the destination receives them, it replies with Echo Reply. This confirms: (1) The network path works, (2) The destination is alive, (3) Your lab is correctly configured. Think of it as knocking on a door and hearing "Come in!"

✅ Success Criteria: You see replies from Metasploitable's IP address. This means your isolated lab network is ready for attacks!

💾 Understanding Snapshots

🤔 WHAT is a Snapshot?

A snapshot is like a "save point" in a video game. It captures the complete state of your virtual machine — the operating system, installed software, files, and even the current memory (RAM). You can instantly revert back to this saved state at any time.

❓ WHY are Snapshots Critical?
  • Recovery from failures: Try an exploit that crashes Metasploitable? Just restore snapshot — back to clean state in seconds.
  • Multiple save points: Save "Before scanning", "After exploit", "Clean install" — jump between them freely.
  • Fearless learning: Experiment without worry. Break things on purpose! You can always revert.
  • Time savings: Instead of reinstalling Linux after a mistake, restore snapshot (5 seconds vs 30 minutes).
🔧 HOW to Use Snapshots Effectively
  • Baseline snapshot: Take one right after installation — clean, working system
  • Pre-attack snapshot: Before trying a new exploit, save current state
  • Post-config snapshot: After setting up tools, save so you don't reconfigure
  • Name clearly: "Kali_Fresh_Install", "MSF_Before_Exploit", "Post_Nmap_Scan"
⚠️ Important Note: Snapshots take disk space. Delete old ones when you don't need them. Never rely on snapshots as your ONLY backup — they can get corrupted.

🔧 Lab 2: Create Snapshots

What we're doing: Saving the current state of your VMs so you can always revert to a clean, working configuration.

📸 Taking a Snapshot (VirtualBox)
  1. Select your VM (Kali or Metasploitable) in VirtualBox main window
  2. Click the "Snapshots" tab (right side, next to Details)
  3. Click the camera icon "Take" (or Machine → Take Snapshot)
  4. Name your snapshot: "Clean Install - Kali" or "Before First Exploit"
  5. Add description (optional): "Fresh Kali install, updated packages, ready for labs"
  6. Click OK — wait 10-30 seconds for completion
  7. To restore: Select snapshot → Click "Restore" → Confirm

How to take and restore snapshots in VirtualBox

📸 Taking a Snapshot (macOS)
  1. UTM: Select VM → Click "Save" (disk icon with +) → Name snapshot → Save
  2. VMware Fusion: VM menu → Snapshots → Take Snapshot
  3. VirtualBox (Intel): Same as Windows steps
  4. Name clearly and add description
  5. To restore: Select snapshot → Restore
🎥 Video Tutorial
📸 Taking a Snapshot (Linux)
  1. VirtualBox: Same as Windows steps
  2. KVM (virt-manager): Select VM → Manage VM Snapshots (camera icon) → Create
  3. Command line: virsh snapshot-create-as [VM-name] --name "snapshot-name"
  4. List snapshots: virsh snapshot-list [VM-name]
  5. Restore: virsh snapshot-revert [VM-name] [snapshot-name]
