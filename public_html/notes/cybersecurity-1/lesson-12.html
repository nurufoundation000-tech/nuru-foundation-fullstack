🧪 WEEK 4 — CLASS 3

PRACTICAL: USERS, GROUPS & PERMISSIONS

The Mastery Class: Today you learn how to control exactly who can enter the computer and what they can touch.

⭐ 1. Today’s Mission

Security is about Access Control. Today, you are the Gatekeeper.

  • 👤 Manage Users: Create and switch between different people.
  • 👥 Manage Groups: Put people into teams to give them group powers.
  • 🔐 Master Permissions: Lock files so only specific people can see them.

This is exactly how real system administrators protect big servers!

👤 2. Lab 1 — User Management

Think of your Linux system like a house. We need to create a "Guest Room" for a new person.

sudo adduser student1

What happens? Linux asks you to set a password for the new person. Type it carefully (you won't see stars as you type!).

To "log in" as that person in your terminal, use:

su student1

Check your terminal prompt—it should now say "student1" instead of your name!

👥 3. Lab 2 — Group Management

It's hard to give keys to 100 people one by one. Instead, we create a Club (Group) and give the keys to the club!

Step 1: Create the "Cyber" Club:

sudo groupadd cybersecurity

Step 2: Put student1 into the club:

sudo usermod -aG cybersecurity student1

Now, anything the cybersecurity group is allowed to do, student1 can do too!

🔐 4. Lab 3 — Permissions Practice

First, let's look at the secret "Security Code" of your files. Type this:

ls -l

You will see something like: -rwxr-xr--

  • First 3 (rwx): What the Owner can do.
  • Middle 3 (r-x): What the Group can do.
  • Last 3 (r--): What Everyone else can do.

🔢 The "Magic Numbers" of chmod

We use numbers to set permissions quickly. Here is the Cheat Code:

NumberPower LevelWhat it means
7Full PowerRead + Write + Run
5Watch & RunRead + Run (No changing!)
0No PowerLocked out!

Try these:

chmod 700 secret.txt ← (Only Owner has power, others get 0)
chmod 755 script.sh ← (Owner full power, others can only watch/run)

🤝 Giving a File Away (chown)

Sometimes you create a file, but you want someone else to be the boss of it.

sudo chown student1 file.txt

chown stands for CHange OWNer. Now, student1 is the owner of that file and can decide its permissions.

Security Tip: Attackers love to find files that are 777 (Everyone has full power). Never make a sensitive file 777!

🧠 5. Security Discussion

Let's think like a Defender:

  • ❓ Why stop Root access? If a hacker gets into a Root account, they can delete the entire computer in one second.
  • ❓ What is a misconfiguration? It's when a lazy admin leaves a "Secret" file with "Read" permissions for everyone.
  • ❓ Damage Control: Permissions ensure that if one user gets hacked, the rest of the system stays safe!
