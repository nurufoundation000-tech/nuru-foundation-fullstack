🧪 WEEK 6 — CLASS 2

PRACTICAL LAB

Observing Malware Behavior (Safe Environment)

⚠️ "First, do no harm."
In this lab, we learn how to watch a predator without getting bitten.

🚨 1. VERY IMPORTANT SAFETY RULE

DO NOT download real malware on your personal computer.

Running malicious code outside of a "Sandbox" can destroy your data and infect your entire home network.

Today we use only:

  • ✅ Safe Videos: Controlled recordings of malware in action.
  • ✅ Simulated Environments: "Fake" malware that shows the effect without the damage.
  • ✅ Cloud Sandboxes: Tools like Any.Run where malware runs on a remote server, not your PC.

⭐ 2. Today’s Mission

As a Security Technician, you must be able to spot the "Symptoms" of an infection.

Your goals for this lab:

  • ✔ Identify the "Trigger": What starts the attack?
  • ✔ Watch the "Propagation": How does it move through the system?
  • ✔ Identify the "Payload": What is the final damage (Encryption, Spying, Theft)?
  • ✔ Think like an Analyst: Connect the dots between system changes and hacker goals.

🎥 3. Lab 1 — Malware Behavior Observation

We will now watch a controlled simulation of a Trojan infection.

🔍 What to Observe (The Checklist):

  • Execution: Did the malware start immediately, or did it wait?
  • System Load: Does the CPU spike to 100%? Do the fans get loud?
  • File Modification: Are files disappearing or getting weird new names?
  • Network Activity: Is the computer trying to connect to a random IP address in another country?

🧠 4. Behavior Analysis (Technician Level)

In the industry, we look for IOCs (Indicators of Compromise). These are the "digital footprints" left by the enemy.

📂 File Changes
Creation of hidden folders (e.g., in AppData) or encrypted .tmp files.
⚙️ System Changes
New "Startup" entries in Task Manager. The malware wants to survive a reboot!
🌐 Network Behavior
Connecting to a "Command & Control" (C2) server to receive instructions.

This process is called Dynamic Analysis (analyzing code while it is running).

🔥 5. Lab 2 — Simulated Malware Actions

The instructor will now demonstrate three "Fake" malware scenarios:

  • Scenario A (Ransomware): A script that renames all your desktop pictures to .LOCKED and opens a Notepad "Ransom Note."
  • Scenario B (Spyware): A program that records every word you type into a secret log.txt file on the desktop.
  • Scenario C (Trojan): A "Calculator" app that secretly opens a CMD window in the background to download more files.

🎯 Lab Goal: Malware is not Magic

Students often think malware is a "ghost" in the machine. It isn't. It follows patterns.

The Standard Attack Pattern:

  1. Ingress: Entering the PC (Phishing email).
  2. Staging: Unpacking itself into a hidden folder.
  3. Persistence: Adding itself to the "Auto-start" list.
  4. Execution: Performing the bad deed (Stealing or Locking).

📌 6. Class Activity: The Malware Diary

Based on the simulations you just watched, write a short "Incident Report":

Incident #001 Report:
1. Trigger: User opened "Invoice.pdf.exe"
2. First Sign: Mouse started lagging, CPU hit 90%.
3. System Change: New file appeared in Startup folder.
4. Result: Desktop wallpaper changed to a warning message.

"If you can describe it, you can defend against it."

🏁 End of Class 2 Outcome

You have moved from being a victim to being a Witness. You can now:

  • ✅ Observe malware safely without risking your own equipment.
  • ✅ Use professional terms like IOCs and Persistence.
  • ✅ Explain the step-by-step behavior of common infections.
  • ✅ Spot the "Warning Signs" of a real-world compromise.
