Data Pipeline Architecture

Learning Objectives

  • Learn common data pipeline architecture patterns
  • Understand cloud data platforms and services
  • Implement pipeline automation and scheduling
  • Apply best practices for reliable data pipelines

Architecture Patterns

Modern data pipelines follow different architectural patterns depending on requirements for latency, volume, and processing complexity.

Lambda Architecture

Batch Layer → Speed Layer → Serving Layer

Combines batch processing (for accuracy) with stream processing (for speed). Good for systems needing both real-time and historical data.

Kappa Architecture

A simplified approach that treats all data as streams. Uses a log-based message broker (like Kafka) as the source of truth, eliminating the batch layer.

Data Lake Architecture

Stores raw data in its native format. Uses schemas-on-read approach, allowing flexibility in how data is processed later.

Modern Trend: Many organizations are moving toward "lakehouse" architectures that combine the flexibility of data lakes with the management features of data warehouses.

Cloud Data Platforms

Cloud platforms provide managed services that simplify data infrastructure.

AWS Data Services

  • S3: Object storage for data lake
  • Redshift: Data warehouse
  • Kinesis: Stream processing
  • Glue: ETL service
  • Athena: Query service for S3

Google Cloud Platform

  • BigQuery: Serverless data warehouse
  • Dataflow: Stream and batch processing
  • Pub/Sub: Messaging service
  • Cloud Storage: Object storage

Azure

  • Synapse: Analytics service
  • Data Factory: ETL/orchestration
  • Blob Storage: Object storage
  • Stream Analytics: Stream processing

Pipeline Automation

Automation ensures pipelines run reliably on schedule without manual intervention.

Workflow Orchestration

# Apache Airflow DAG example from airflow import DAG from airflow.operators.python import PythonOperator with DAG('data_pipeline', schedule_interval='@daily') as dag: extract = PythonOperator( task_id='extract_data', python_callable=extract_from_db ) transform = PythonOperator( task_id='transform_data', python_callable=clean_and_transform ) load = PythonOperator( task_id='load_to_warehouse', python_callable=load_to_dw ) extract >> transform >> load

Key Orchestration Tools

  • Apache Airflow: Open-source, widely used
  • Prefect: Python-native, modern interface
  • Dagster: Built for production
  • Apache NiFi: Visual data flow design
Best Practice: Design pipelines to be idempotent—running them multiple times should produce the same result. This enables safe retries and re-runs.

Reliability Best Practices

Building reliable pipelines requires addressing common failure modes.

Error Handling

  • Implement retry logic for transient failures
  • Add dead-letter queues for failed records
  • Log errors with sufficient context for debugging
  • Set up alerts for pipeline failures

Monitoring

  • Track pipeline execution times
  • Monitor data quality metrics
  • Alert on data freshness issues
  • Log data volume changes

Data Quality

  • Validate schema at pipeline entry/exit points
  • Check for null values and duplicates
  • Implement data freshness checks
  • Create data lineage tracking
Important: Never silently drop failed records. Implement proper error handling and logging so issues can be investigated and resolved.

Summary

  • Lambda and Kappa architectures offer different approaches to batch vs. stream processing
  • Cloud platforms provide managed services that simplify infrastructure
  • Workflow orchestration tools automate pipeline execution
  • Reliability requires proper error handling, monitoring, and data quality checks
