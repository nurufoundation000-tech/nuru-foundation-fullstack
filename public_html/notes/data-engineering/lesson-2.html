Working with Databases

Learning Objectives

  • Master SQL fundamentals for data querying
  • Understand database design principles
  • Learn about different types of data storage
  • Apply best practices for database management

SQL Fundamentals

SQL (Structured Query Language) is the standard language for working with relational databases. It's essential for data engineers to retrieve, manipulate, and manage data.

Basic Queries

-- Select all columns from a table SELECT * FROM employees; -- Select specific columns SELECT name, email, salary FROM employees; -- Filter with WHERE clause SELECT * FROM employees WHERE department = 'Sales'; -- Sort results SELECT * FROM employees ORDER BY salary DESC;

Aggregation Functions

SELECT COUNT(*) FROM employees; SELECT SUM(sales) FROM orders; SELECT AVG(salary) FROM employees; SELECT MAX(salary), MIN(salary) FROM employees;

Grouping and Filtering

-- Group by department SELECT department, AVG(salary) as avg_salary FROM employees GROUP BY department; -- Filter groups with HAVING SELECT department, COUNT(*) as emp_count FROM employees GROUP BY department HAVING COUNT(*) > 5;

Database Design

Good database design ensures data integrity, efficient queries, and scalability.

Normalization

Normalization organizes data into tables to reduce redundancy:

  • 1NF: No repeating groups, atomic values
  • 2NF: No partial dependencies
  • 3NF: No transitive dependencies

Keys and Relationships

Key Type Description
Primary Key Unique identifier for each record
Foreign Key Links to another table's primary key
Composite Key Combination of multiple columns
Index Improves query performance
Tip: Index columns that are frequently used in WHERE clauses and JOIN operations. However, too many indexes can slow down INSERT/UPDATE operations.

Types of Data Storage

Data engineers work with various storage systems depending on use cases:

Relational Databases

  • PostgreSQL: Feature-rich open-source
  • MySQL: Popular for web applications
  • SQL Server: Enterprise-grade from Microsoft

NoSQL Databases

  • MongoDB: Document database
  • Cassandra: Wide-column store
  • Redis: In-memory key-value store
  • Neo4j: Graph database

Cloud Data Warehouses

  • Snowflake: Multi-cluster architecture
  • BigQuery: Serverless from Google
  • Redshift: AWS data warehouse
Choice Guide: Use relational databases for structured data and complex queries. Choose NoSQL for flexible schemas and high write throughput. Use data warehouses for analytics and BI workloads.

Best Practices

  • Use parameterized queries to prevent SQL injection
  • Implement proper indexing for query performance
  • Regularly backup databases to prevent data loss
  • Monitor query performance and optimize slow queries
  • Use transactions for multi-step operations
  • Implement access controls for security
Security: Never concatenate user input directly into SQL queries. Always use parameterized queries or prepared statements to prevent SQL injection attacks.

Summary

  • SQL is essential for querying and manipulating data in relational databases
  • Database design follows normalization principles to reduce redundancy
  • Different storage types serve different purposes (OLTP vs OLAP)
  • Best practices include indexing, backups, and security measures
