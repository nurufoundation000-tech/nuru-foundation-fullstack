🌐 WEEK 4 — CLASS 1

OPERATING SYSTEMS BASICS

Security Foundation: Understanding the "Boss" of the computer to become a System Defender.

⭐ 1. Today’s Mission

Cybersecurity isn't just about stopping hackers; it's about controlling the rules of the computer.

Today you will learn who can:

  • 📂 Access Files: Who is allowed to open and see data?
  • 🚀 Run Programs: Who is allowed to start software?
  • ⚙️ Change Settings: Who can mess with the rules?

By the end, you'll think like a system defender!

💻 2. What Is an Operating System (OS)?

The OS is the BOSS of the computer. It makes all the big decisions.

The Boss controls:

  • 🔌 Hardware: The Brain (CPU), Memory (RAM), and Storage (Disk).
  • 💾 Software: All the apps you use.
  • 👤 Users: Keeping people's accounts separate.
  • 📁 Files: Organizing all your data.
  • 🛡️ Security: Setting the rules of the house.

Examples: Windows, Linux, and macOS. Without an OS, a computer is useless.

🪟 3. Windows Security Introduction

Windows is the most used OS, which makes it the #1 target for hackers.

  • 👤 User Accounts: Each user has a name, password, and "Access Level." Administrators have full control—this is what hackers want!
  • 🛡️ Windows Defender: Built-in police that hunts for viruses, ransomware, and spyware. Keep it updated!
  • 🧱 Windows Firewall: Controls traffic. It blocks hackers from entering through the internet.
  • 🔄 Windows Updates: Fixes "holes" (vulnerabilities). Unpatched systems are easy targets.

🐧 4. Linux Security Introduction

Pros love Linux because it is strict and transparent (Open-Source). Examples: Ubuntu (Easy) or Kali Linux (Security testing).

  • 👑 Root vs Normal User: The Root user is the "System God" with total power. Normal users have very limited access.
  • ⚠️ Safety Rule: Never work as "Root" all the time! It's too dangerous.
  • 🛠️ Sudo Command: Type sudo before a command to get temporary admin power. It limits the damage if something goes wrong.

🐧 Linux Common Directories

You must know these folders to find where the "secrets" and "tools" are kept:

  • 📂 /home → Where your personal user files live.
  • ⚙️ /etc → Where the Settings (configuration) are.
  • 📜 /var → Where the Logs (history of what happened) are.
  • 🔧 /bin → Where the System Programs live.
  • 👑 /root → The private folder for the "System God."

🔑 6. Users, Groups & Permissions

Security is simply asking: "Who can do what?"

  • 👤 Users: Each person has a name, password, and home folder.
  • 👥 Groups: Users are put in teams (like Admins, Staff, or Students). It's easier to give rules to a whole team at once!

The 3 Powers (Permissions):

1. 📖 Read (r): Can you look at the file?
2. ✍️ Write (w): Can you change or delete it?
3. 🏃 Execute (x): Can you run it as a program?

🔐 Decoding the Secret Code

In Linux, permissions look like a string of letters. This is core security knowledge!

- rwx r-x ---

How to read it:

  • 🥇 Owner (rwx): Can Read, Write, and Run.
  • 🥈 Group (r-x): Can Read and Run, but NOT change.
  • 🥉 Others (---): No access at all! This is very safe.

🧠 7. Why This Matters

When hackers attack, they have 4 main goals:

  • ❌ Gain Admin/Root access (to become the "Boss").
  • ❌ Steal important files.
  • ❌ Install Malware (viruses).
  • ❌ Disable Security Tools (like the Firewall).

If you master OS security, you can build a defense that is too hard for them to break!
