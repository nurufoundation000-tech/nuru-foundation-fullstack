OSINT & Nmap Scanning

🎯 Objective: Gather information about a target using passive and active techniques

Today, you'll run real OSINT commands and Nmap scans against safe, authorized targets to discover what information is exposed.

⚠️ Legal Notice: All scans in this class must be performed against YOUR own lab machines (Metasploitable) or authorized test targets only. Scanning random websites or IPs without permission is illegal.

🔧 Lab 1: OSINT on a Domain

🤔 WHAT are we doing in this lab?

We will use command-line tools to query public databases and DNS servers. These queries are completely passive — the target domain never knows we're looking. Think of it as reading public records and phone books.

Safe target for practice: We'll use example.com (a real domain owned by IANA specifically for documentation and testing). You can also practice on your own domain if you own one.

📋 Step 1: WHOIS Lookup

What this command does: Queries the global WHOIS database to retrieve registration information about a domain name.

whois example.com

What to look for in the output:

  • Registrant: Who owns the domain?
  • Creation/Expiration dates: When was it registered? When does it expire?
  • Name Servers: Which DNS servers handle this domain?
  • Registrar: Which company sold the domain?

Why this matters: Attackers use WHOIS to identify domain owners for targeted phishing, find expired domains to buy, or discover related domains owned by the same person.

📡 Step 2: DNS Lookup (DIG)

What this command does: Queries DNS servers to retrieve specific records for a domain. DNS is the "phonebook of the internet" — it translates human-readable names (google.com) to IP addresses.

dig example.com

What to look for:

  • A Record: IPv4 address of the domain
  • AAAA Record: IPv6 address
  • MX Record: Mail servers (target for email attacks)
  • NS Record: Name servers
  • TXT Record: Text information (SPF, DKIM, verification codes)

🔧 Lab 1: OSINT on a Domain (Continued)

🔍 Step 3: Identify Subdomains

What are subdomains? Subdomains are prefixes added to a domain name (e.g., admin.example.com, mail.example.com, dev.example.com). They often reveal hidden or internal services.

Manual discovery methods:

  • Look at Name Servers in WHOIS output — sometimes they hint at subdomain structure
  • Check Related Domains — domains owned by the same registrant
  • Try common subdomains: www, mail, ftp, admin, dev, test, api, vpn, remote
# Quick test for common subdomain
dig admin.example.com

More advanced (future class): Tools like sublist3r, amass, or dnsrecon automate subdomain discovery.

🧾 Expected Outcome
✅ After completing Lab 1, you will have:
  • Basic domain information (owner, registrar, dates)
  • DNS records (A, MX, NS, TXT records)
  • Public data about the target without ever contacting their servers
  • Understanding of what information is publicly available about ANY domain
# Example DIG output interpretation:
;; ANSWER SECTION:
example.com. 86400 IN A 93.184.216.34
# This means: example.com resolves to IP 93.184.216.34
💡 Pro Tip: Use dig +short example.com to get just the IP address without extra details. Use dig example.com ANY to get ALL record types (though many DNS servers restrict this).

🔧 Lab 2: Nmap Advanced Scans

🤔 WHAT are we doing in this lab?

We will use Nmap (Network Mapper) to actively scan a target and discover open ports, running services, and potential vulnerabilities. Unlike OSINT, this IS active reconnaissance — the target will see these probes in their logs.

Safe target for practice: We will scan YOUR Metasploitable VM (192.168.56.102) — a deliberately vulnerable machine designed for this exact purpose.

🔍 Scan 1: Basic Port Scan

What this does: Scans the most common 1,000 ports to see which are open and listening for connections.

nmap 192.168.56.102

What to observe:

  • Open ports: 21 (FTP), 22 (SSH), 80 (HTTP), 443 (HTTPS), 445 (SMB), etc.
  • Service names: What each port typically runs
  • State: "open" means the port accepts connections
# Example output:
21/tcp open ftp
22/tcp open ssh
80/tcp open http
🔧 Scan 2: Service Version Detection

What this does: Connects to open ports and analyzes responses to determine exact software and version numbers.

nmap -sV 192.168.56.102

Why version detection is critical: Knowing the exact version tells you which known vulnerabilities exist. For example, Apache 2.4.49 has a critical path traversal vulnerability that 2.4.50 fixed.

# Example output with versions:
21/tcp open ftp vsftpd 2.3.4
22/tcp open ssh OpenSSH 4.7p1
80/tcp open http Apache httpd 2.2.8

🔧 Lab 2: Nmap Advanced Scans (Continued)

⚠️ Scan 3: Vulnerability Scripts

What this does: Runs Nmap's built-in vulnerability detection scripts against discovered services. These scripts check for known CVEs and common misconfigurations.

nmap --script vuln 192.168.56.102

What to look for:

  • Script output showing potential vulnerabilities
  • CVE identifiers (e.g., CVE-2011-2523)
  • Warning messages about insecure configurations

Note: Vulnerability scripts can be noisy and may crash unstable services. Use cautiously.

🛡️ Scan 4: Firewall Evasion Techniques

What this does: Uses packet fragmentation to bypass simple firewalls and intrusion detection systems.

nmap -f 192.168.56.102

How it works: The -f flag splits TCP packets into smaller fragments. Some firewalls only inspect the first fragment and let others pass through.

Other evasion techniques:

  • -D RND:10 → Decoy scan (masks your real IP with decoys)
  • -T0 → Paranoid slow scan (avoids IDS detection)
  • --source-port 53 → Spoof source port as DNS (some firewalls trust port 53)
  • --mtu 24 → Custom MTU size for fragmentation
🧾 Expected Outcome of Lab 2:
  • ✅ Open ports identified (which services are reachable)
  • ✅ Running services detected (with version numbers)
  • ✅ Possible vulnerabilities listed (from vulnerability scripts)
  • ✅ Understanding of how to adjust scans for different network conditions

📚 Nmap Command Reference & Best Practices

🔧 Useful Nmap Options
  • -sS → SYN stealth scan (faster, less logging)
  • -sT → TCP connect scan (more reliable, more detectable)
  • -sU → UDP scan (slow, but finds UDP services)
  • -p- → Scan ALL 65535 ports (takes time!)
  • -p 22,80,443 → Scan specific ports only
  • -T4 → Aggressive timing (faster scans)
  • -A → Aggressive scan (OS detection + version + traceroute + scripts)
  • -oN output.txt → Save results to file
⚠️ Nmap Best Practices
  • Always get permission first — Scanning without authorization is illegal
  • Start slow — Use -T2 or -T3 on production networks
  • Save your output — -oA scan_name saves in all formats
  • Know your target — Scanning local lab vs internet target is very different
  • Understand what you're doing — Some scans can crash vulnerable services
# Comprehensive scan example:
nmap -sS -sV -O -p- --script vuln -T4 192.168.56.102 -oA metasploitable_scan
📌 REMEMBER: In your lab, Metasploitable is DESIGNED to be vulnerable. On real networks, aggressive scanning can trigger security alerts, crash services, and get your IP blocked or reported. Always practice in isolated environments first.
