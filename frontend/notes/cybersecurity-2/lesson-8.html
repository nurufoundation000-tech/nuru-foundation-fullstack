🔧 Lab 1: Install Kali Linux (Virtual Machine)

📋 Step-by-Step (Windows)
  1. Download and install VirtualBox from official website
  2. Open VirtualBox → Click "New" → Name: Kali Linux → Type: Linux → Version: Debian (64-bit)
  3. RAM: 4096 MB (4 GB) minimum | Disk: 40 GB
  4. Create → VDI → Dynamically allocated → 40 GB
  5. Settings → Storage → Empty CD/DVD → Choose disk → Select Kali ISO
  6. Start VM → Follow graphical installer → Set username/password
  7. After installation, remove ISO and reboot
🎥 Video (Windows)

Recommended: NetworkChuck's Kali setup tutorial

📋 Step-by-Step (macOS)
  1. Intel Mac: Download VirtualBox. Apple Silicon (M1/M2/M3): Download UTM from mac.getutm.app
  2. For UTM: Click "Create VM" → Linux → Browse for Kali ARM ISO
  3. Allocate RAM: 4 GB (4096 MB) | Disk: 40 GB
  4. Complete installation and boot into Kali
  5. For VirtualBox: Same as Windows steps above
🎥 Video (macOS)

For Apple Silicon (M1/M2) users - UTM setup

📋 Step-by-Step (Linux)
  1. Install VirtualBox: sudo apt install virtualbox or use your package manager
  2. Alternative: Use KVM/QEMU for better performance: sudo apt install qemu-kvm virt-manager
  3. Create new VM in VirtualBox or virt-manager with 4GB RAM, 40GB disk
  4. Attach Kali ISO and boot
  5. Follow standard Debian installation
  6. Reboot and login

🔧 Lab 2: Install Metasploitable

  1. Download Metasploitable 2 ZIP from SourceForge
  2. Extract ZIP to a folder (e.g., C:\VMs\Metasploitable)
  3. Open VirtualBox → File → Import Appliance
  4. Browse to the extracted .ovf file
  5. Click Import → Wait for import to complete
  6. Start VM → Login: msfadmin / msfadmin

🌐 Lab Network Configuration

  1. Open VirtualBox → File → Preferences → Network → Host-Only Networks
  2. Click "+" to create vboxnet0 (usually 192.168.56.1/24)
  3. Select Kali VM → Settings → Network → Adapter 1 → Host-Only → Name: vboxnet0
  4. Select Metasploitable VM → Same: Host-Only → vboxnet0
  5. Start both VMs → Run ip a → IPs should be 192.168.56.x
✅ Test Connectivity
ping 192.168.56.102
nmap -sV 192.168.56.102
  1. For VirtualBox on Intel: Same as Windows steps above
  2. For UTM: Create a shared network between VMs
  3. Go to UTM Preferences → Network → Create "Shared Network"
  4. Assign both Kali and Metasploitable to that same shared network
  5. Boot both → Check IPs with ip a
  1. For VirtualBox: Same as Windows steps
  2. For KVM: Use virsh net-create or virt-manager → Add isolated network
  3. Both VMs on same virtual network → Usually 192.168.122.x range
  4. Test with ping and nmap
⚠️ Important: Never put Metasploitable on Bridged or NAT. Keep it isolated to Host-Only or an internal network!

🔧 Lab 3: Access DVWA

📋 Steps to Access
  1. Start Metasploitable VM
  2. Find Metasploitable IP: ip a (look for eth0 or ens33)
  3. In Kali, open Firefox → http://[Metasploitable-IP]/dvwa
  4. Login: admin / password
  5. Start practicing: SQL Injection, XSS, Brute Force

🧾 Outcome & Troubleshooting

✅ Successful lab setup means:
  • ✔ Kali Linux installed and booting
  • ✔ Metasploitable running and reachable via ping
  • ✔ DVWA accessible from Kali's browser
❌ Can't ping Metasploitable
  • Both VMs on same network?
  • Run sudo dhclient eth0 on Metasploitable
  • Disable firewall: sudo ufw disable
❌ DVWA shows "Unable to connect"
  • Check IP with ip a on Metasploitable
  • Web server status: sudo systemctl status apache2
  • Try http://[IP]/dvwa/ (trailing slash)
