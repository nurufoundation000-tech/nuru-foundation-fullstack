🧠 1. Deep Understanding of the OSI Model

The OSI Model (Open Systems Interconnection) describes how data moves from one device to another across a network in 7 logical layers.

Mental Model

Think of sending a package: you write a message, wrap it, label it, choose a route, and hand it over to a delivery service, which finally transports it physically.

Each layer of OSI is responsible for one part of this “delivery pipeline”. When something breaks or gets attacked, we can often map it to a specific layer.

Upper Layers (Closer to the User)
  • Application Layer – Where users interact with the network (web browsers, email clients, file transfer apps).
  • Presentation Layer – Formats, compresses, and encrypts data so that applications can understand each other.
  • Session Layer – Opens, maintains, and closes communication sessions between devices.
Lower Layers (Closer to the Wire)
  • Transport Layer – Controls how data is sent (reliable vs fast delivery, TCP vs UDP).
  • Network Layer – Handles logical addressing and routing (IP addresses, routers).
  • Data Link Layer – Handles communication between devices on the same local network (MAC addresses, switches).
  • Physical Layer – Deals with electricity, light, radio waves, cables, and WiFi signals.

Key idea: Every cyber attack touches at least one OSI layer. Knowing the layers makes you better at locating and defending against attacks.

🔐 OSI Model from a Security View

From a security perspective, we think of the OSI model as 7 “attack surfaces”. Different types of attacks live at different layers.

Common Attack Examples
  • Application Layer – Web attacks such as SQL Injection, Cross-Site Scripting (XSS), and insecure APIs.
  • Transport Layer – SYN flood attacks that abuse the TCP handshake to overload a server.
  • Network Layer – IP spoofing where an attacker forges source IP addresses.
  • Data Link Layer – ARP spoofing to redirect traffic inside a local network.
Important Note

Cyber attacks are not random; they usually target a specific layer with a specific weakness.

If you can say “this is a Layer 7 problem” or “this is a Layer 2 attack”, you are already thinking like a security analyst.

✨ 2. Packet Flow Inside a Network

A packet is a small unit of data sent across a network. It is like one “envelope” in a larger conversation.

Packet Basics
  • Source address – Where the packet is coming from (IP).
  • Destination address – Where the packet is going (IP).
  • Data (payload) – The actual content being delivered (HTTP request, email content, etc.).
From Browser to Server

Example: A user enters google.com in the browser.

  • DNS converts the human-readable name into an IP address.
  • The client and server agree to communicate using TCP.
  • Packets are then created, addressed, and sent through routers on the internet.

For defenders: If you can follow a packet hop-by-hop, you can also imagine how an attacker might intercept or modify it.

📡 TCP Three-Way Handshake

Before two devices exchange data using TCP, they first “shake hands” to agree on how to communicate reliably.

Three Steps
1
SYN – Client sends a request to connect.
2
SYN-ACK – Server acknowledges and agrees to connect.
3
ACK – Client confirms the connection is established.

After this three-way handshake, data can flow in both directions using the agreed sequence numbers.

Security Angle

Attackers can abuse this handshake with attacks like SYN floods, where they start thousands of connections but never finish them, overloading the server.

⚠️ Security Risks in Packet Flow

Once you understand how packets move, you can see where attackers may insert themselves in the path.

Common Risks
  • DNS spoofing – DNS responses are manipulated so users are sent to fake websites.
  • Traffic interception – Man-in-the-Middle (MITM) attacks where an attacker sits between the user and the destination.
  • Packet capture – Attackers sniff and analyze traffic to steal credentials or session tokens.
Reality Check

Network communication is always a target. Every time data moves, there is a chance for someone to listen, modify, or block it.

✨ 3. Important Network Protocols

Some protocols are especially important in both normal operation and in many attacks.

ARP – Address Resolution Protocol

ARP maps IP addresses to MAC addresses within a local network.

Informally, ARP asks: “Who has this IP? Tell me your physical (MAC) address.”

Weakness & Attack

ARP does not verify identity. An attacker can respond pretending to be another device – this is ARP spoofing.

DHCP – Dynamic Host Configuration Protocol

DHCP automatically assigns IP addresses and other network settings to devices when they join a network.

Risk

A fake DHCP server can give wrong IP/gateway/DNS settings and silently redirect traffic through the attacker.

ICMP – Internet Control Message Protocol

ICMP is used for network diagnostics, such as the ping command.

Risk

ICMP can be abused in flooding attacks or to map out live hosts and network structure.

BGP – Border Gateway Protocol

BGP decides how data is routed between large networks (ISPs) across the internet.

Risk

Attackers can perform BGP hijacking, redirecting large amounts of traffic to the wrong location.

🔁 NAT – Network Address Translation

NAT converts private IP addresses (inside a local network) into a public IP address that is visible on the internet.

How NAT Helps
  • Hides internal IP addresses from the public internet.
  • Allows many devices in a home or office to share a single public IP.
  • Adds a basic layer of isolation between internal devices and the outside world.
But Be Careful

NAT is not a full security solution. It hides internal addresses, but it does not stop application-level attacks or prevent exploitation of vulnerable devices.

Takeaway: NAT helps, but you still need firewalls, secure configurations, and good application security.

✨ 4. How Attackers Use Network Behavior

Attackers exploit the “trust” built into protocols and devices. They often do not attack the hardware, but the assumptions behind the communication.

Common Techniques
  • Packet sniffing – Capturing traffic to read unencrypted data.
  • ARP poisoning – Corrupting ARP tables to redirect traffic.
  • Man-in-the-Middle (MITM) – Sitting between two parties to intercept and modify data.
  • Session hijacking – Stealing a valid session token to take over a logged-in session.
  • Traffic replay – Recording valid traffic and replaying it later to bypass checks.
Example – Public WiFi
  • An attacker connects to the same public WiFi as Alex Chen’s son.
  • They perform ARP spoofing to become the “middle” between the victim and the router.
  • They capture unencrypted logins, cookies, or other sensitive data while traffic passes through them.

Mindset shift: Every “normal” network behavior (ARP requests, DHCP offers, TCP handshakes) can also become a weapon in the wrong hands.
