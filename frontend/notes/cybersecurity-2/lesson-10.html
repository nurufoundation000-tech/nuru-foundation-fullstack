Reconnaissance & Footprinting

📘 CYBERSECURITY LEVEL 2 – WEEK 4 | CLASS 1

Before any attack or defense, you must understand the target. Reconnaissance is the art of gathering intelligence — legally and ethically.

🎯 What you'll learn today:
  • Passive vs Active Reconnaissance — the difference between listening and knocking
  • OSINT (Open Source Intelligence) — finding public information about any target

1. Passive Reconnaissance (OSINT)

🤔 WHAT is Passive Reconnaissance?

Passive reconnaissance is the process of collecting information about a target WITHOUT directly interacting with it. You never send packets to the target's systems. Instead, you gather data from public sources, like a detective reading newspapers and watching from across the street.

Key principle: The target has NO idea you're researching them.

📚 What is OSINT?

OSINT (Open Source Intelligence) refers to data collected from open and public sources that anyone can legally access.

Sources include:

  • Websites and blogs
  • Social media (LinkedIn, Twitter, Facebook)
  • Public records and government databases
  • Search engines (Google, Bing, DuckDuckGo)
  • DNS records (who owns a domain)
  • GitHub repositories (accidentally exposed code)
  • Wayback Machine (old versions of websites)
📋 Information You Can Gather
  • Domain names — what websites does the company own?
  • IP addresses — where are their servers located?
  • Email addresses — employee emails (e.g., john.doe@company.com)
  • Subdomains — admin.company.com, mail.company.com, dev.company.com
  • Company details — employees, locations, partners, technologies used
  • Breached credentials — have employees' passwords been leaked?
✅ Advantages of Passive Reconnaissance:
  • No direct contact with target — completely stealthy
  • Hard to detect — no logs are created on the target's systems
  • Legal when done properly — you're just using public information
  • Often reveals surprising details employees accidentally exposed
💡 Real Example: Searching "example.com" on LinkedIn can reveal employee names and job titles. Combining that with email pattern "first.last@example.com" gives you valid email addresses for phishing tests or social engineering simulations.

2. Active Reconnaissance

🤔 WHAT is Active Reconnaissance?

Active reconnaissance involves DIRECT interaction with the target system. You send packets, make requests, and analyze the responses. Think of it as knocking on doors to see who answers — the target KNOWS someone is checking.

🔧 Common Techniques
  • Port scanning — which network ports are open? (e.g., port 80 for web, port 22 for SSH)
  • Service detection — what software is running? (Apache 2.4, OpenSSH 7.2, etc.)
  • Network mapping — what devices are on the network? How are they connected?
  • Banner grabbing — connecting to a service and reading its welcome message (often reveals version info)
  • Vulnerability scanning — checking for known weaknesses
⚡ Characteristics to Understand
  • Faster and more detailed — you get specific, confirmed data
  • Detectable by security systems — firewalls and IDS/IPS log your probes
  • Requires controlled environments — in real pentesting, you need written permission!
  • Can trigger alarms — aggressive scanning might get your IP blocked
📊 Passive vs Active — The Key Difference:
PassiveNo packets sent to target → Undetectable → Slower, less detailed
ActiveDirect interaction → Detectable → Fast, detailed results

Professional approach: Start with passive (stay quiet). Once you have permission, move to active scanning.

3. Attack Surfaces

🤔 WHAT is an Attack Surface?

The attack surface is the total number of entry points an attacker can use to access a system. Every open port, running service, web form, API endpoint, and even human employee is part of the attack surface.

Simple analogy: A house with 10 doors and 20 windows has a larger attack surface than a house with 1 door and 2 windows. More entry points = more ways to break in.

🌐 Network Attack Surface
  • Open ports (SSH, HTTP, RDP, etc.)
  • Running services (web servers, databases, FTP)
  • Network protocols (SMB, SNMP, Telnet)
  • VPN entry points
💻 Application Attack Surface
  • Web applications and forms
  • APIs (REST, GraphQL, SOAP)
  • Mobile apps and their backends
  • Login pages, search boxes, file uploads
👤 Human Attack Surface
  • Employees (social engineering targets)
  • Contractors and third parties
  • Customer support staff
  • Anyone with access to systems
📌 Key Security Principle: The larger the attack surface, the higher the risk. Reducing attack surface is a fundamental security strategy — close unnecessary ports, disable unused services, remove unneeded accounts.
💡 Why This Matters for Reconnaissance: Your job as a security professional is to discover the attack surface BEFORE attackers do. Reconnaissance finds all the doors. Then you help close the unnecessary ones.

4. Reconnaissance Tools (Part 1)

🌐 WHOIS — Domain Registration Lookup

What it does: WHOIS retrieves domain registration information from public databases. Every registered domain has this info (though privacy services can hide some details).

whois example.com

Information revealed:

  • Domain owner (registrant name and organization)
  • Registration and expiration dates
  • Name servers (DNS servers for the domain)
  • Administrative and technical contacts
  • Registrar used

Why attackers want this: Find the person responsible for the domain, then target them with phishing or social engineering.

📡 DIG — DNS Record Query

What it does: DIG (Domain Information Groper) queries DNS servers to retrieve specific records about a domain. DNS is like the phonebook of the internet — it translates names to IP addresses.

dig example.com # Basic query
dig example.com A # IPv4 address
dig example.com MX # Mail servers
dig example.com NS # Name servers
dig example.com TXT # Text records (SPF, DKIM, etc.)

Why attackers want this: Discover subdomains, find mail servers (target for email attacks), identify third-party services.

💡 Try It Yourself (Legally): In your Kali terminal, run whois google.com and dig google.com. These are public queries — completely legal and undetectable by Google.

4. Reconnaissance Tools (Part 2)

🌍 Shodan — The Internet Search Engine

What it does: Shodan scans the entire internet constantly and indexes every connected device. It's like Google, but for devices instead of web pages.

What you can find:

  • Web servers and their software versions
  • Security cameras (many with default passwords!)
  • Industrial control systems (power plants, water facilities)
  • Databases exposed to the internet
  • SSH, RDP, FTP servers
# Example Shodan search queries:
port:22 country:US
Apache 2.4.49
Default password

Access: shodan.io — free tier available, paid for full access.

🔍 Nmap — Advanced Network Scanner

What it does: Nmap (Network Mapper) discovers hosts and services on a network. It's the most essential active reconnaissance tool in cybersecurity.

nmap 192.168.1.1 # Basic scan
nmap -sV 192.168.1.1 # Service version detection
nmap -O 192.168.1.1 # OS detection
nmap -p- 192.168.1.1 # All 65535 ports
nmap --script vuln 192.168.1.1 # Vulnerability scripts
nmap -sS -sV -O 192.168.1.0/24 # Combined scan
📌 Nmap Scan Types Explained:
  • -sV → Version detection (shows what software and version is running)
  • -O → OS detection (tries to identify the operating system)
  • -p- → Scan all 65,535 ports (default scans only 1,000 common ports)
  • --script vuln → Run vulnerability detection scripts against found services
  • -sS → SYN stealth scan (faster, less likely to be logged)
