🔧 Lab 1: Create a Simple Script

We'll build a script from scratch — to understand the anatomy of automation.

📝 STEP 1 — Create the file
nano script.sh

Always use the .sh extension for clarity. nano is beginner-friendly, but you can also use vim or code.

✍️ STEP 2 — Add content (shebang + commands)
#!/bin/bash
echo "System Scan Started"
date

The #!/bin/bash (shebang) tells the system which interpreter to use.

🔐 STEP 3 — Make executable
chmod +x script.sh

Without execute permission, the script is just a text file. chmod +x adds execution rights.

🚀 STEP 4 — Run the script
./script.sh

The ./ means "look in the current directory". Your terminal will then execute each line in order.

✅ Expected output: "System Scan Started" followed by the current date. Simple, but you've just automated your first task.

🔧 Lab 2: Automation Script

Automation shines when you combine multiple commands that you'd otherwise type manually every day.

⚙️ Example: System Quick-Status Script
#!/bin/bash
echo "===== System Quick Report ====="
echo "Listing files in current directory:"
ls -lah
echo ""
echo "Current user:"
whoami
echo ""
echo "Current working directory:"
pwd
  • Why this matters: Instead of typing ls, whoami, pwd separately, you run one script and get a structured report.
  • Extend it: Add df -h for disk usage or free -h for memory — build your own diagnostic toolkit.
💡 Pro tip: Use absolute paths inside scripts if you plan to run them from different locations.
Example: /home/user/scripts/audit.sh instead of relying on relative paths.

🔁 Making Scripts Smarter

Real automation uses variables and basic logic. Let's upgrade our lab script.

📦 Using variables
#!/bin/bash
target_dir="/var/log"
echo "Inspecting $target_dir"
ls $target_dir | head -5

Variables store information and make scripts reusable.

⚖️ Simple conditions
#!/bin/bash
if [ -f "/etc/passwd" ]; then
echo "Password file exists."
else
echo "Missing critical file."
fi

🧠 Why this matters for pentesters: Conditional logic lets you build scripts that react to the environment — for example, checking if a port is open before launching an exploit.

📌 Common Automation Patterns in Cybersecurity

🔄 Loops for reconnaissance
for ip in 192.168.1.{1..254}; do
ping -c 1 $ip | grep "64 bytes"
done

Simple host discovery loop — scans a /24 subnet and shows live hosts.

📂 Batch log analysis
grep "Failed password" /var/log/auth.log | awk '{print $11}' | sort | uniq -c

One-liner that extracts and counts failed SSH login attempts by IP — pure automation gold.

🎯 Outcome mindset: Once you write a script, you never have to manually repeat that task again. This is the difference between a junior and a senior operator.
